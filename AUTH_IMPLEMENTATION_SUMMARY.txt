API CONTRACT IMPLEMENTATION:
===========================
POST /auth/login
  Request: { email, password }
  Response: { token, refreshToken, user }

POST /auth/register
  Request: { email, password, firstName, lastName }
  Response: { token, refreshToken, user }

GET /auth/me
  Header: Authorization: Bearer <token>
  Response: { user }

POST /auth/refresh
  Request: { refreshToken }
  Response: { token, refreshToken }
  Used internally for token refresh on 401

SECURITY FEATURES:
==================
- Automatic token attachment to all API calls
- 401 auto-refresh: tries refresh token, clears on failure, redirects to login
- localStorage isolation: ae_token, ae_refresh, ae_user keys
- Password field has show/hide toggle
- Form validation on client side
- Error messages displayed inline (no sensitive data leaked)
- CSRF-ready structure (add nonce in production)
